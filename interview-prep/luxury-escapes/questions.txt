🌐 SECTION 1 — WEB FUNDAMENTALS (HIGH PRIORITY)
4. What happens when you type a URL?

“When a user enters a URL, the browser first checks cache. If not found, it performs a DNS lookup to resolve the domain into an IP address. Then a TCP connection is established, and if HTTPS is used, a TLS handshake occurs. The browser sends an HTTP request to the server, which processes it and returns a response. The browser then parses HTML into the DOM, CSS into CSSOM, executes JavaScript, and finally renders the page on the screen.”

5. What is DNS?

“DNS stands for Domain Name System. It translates a human-readable domain name like google.com into an IP address that computers use to locate servers. It also uses caching to improve performance.”

6. HTTP vs HTTPS

“HTTP is a protocol for transferring data over the web, but it is not secure. HTTPS is the secure version that uses SSL/TLS encryption to protect data during transmission. It prevents attacks like man-in-the-middle.”

7. What is SSL/TLS?

“SSL/TLS is a security protocol that encrypts data between client and server. It ensures confidentiality, integrity, and authenticity using certificates.”

8. How browser sends and receives request

“The browser sends an HTTP request to the server, including headers and optional body. The server processes it and returns a response with status code and data. The browser then renders the response.”

9. HTTP Status Codes

“HTTP status codes indicate the result of a request. 200 means success, 400 indicates client error, and 500 indicates server error. They help identify where a problem occurred.”

⚡ SECTION 2 — JAVASCRIPT CORE
10. var vs let vs const

“var is function-scoped and can be redeclared, which can lead to bugs. let and const are block-scoped. const cannot be reassigned, while let can. In modern JavaScript, we avoid var and prefer let and const.”

11. What is a Promise?

“A Promise represents an asynchronous operation that can be in pending, fulfilled, or rejected state. It allows handling async operations more cleanly than callbacks.”

12. Async/Await

“Async/await is built on top of Promises and allows writing asynchronous code in a synchronous style, improving readability.”

13. Event Loop

“JavaScript is single-threaded, and the event loop manages asynchronous operations. It processes the call stack and handles tasks from queues like microtasks and macrotasks.”

14. What is closure?

“A closure is a function that has access to its outer scope even after the outer function has finished executing. It is commonly used for data encapsulation.”

15. Hoisting

“Hoisting is JavaScript behavior where variable and function declarations are moved to the top of their scope during execution. Variables declared with var are hoisted differently than let and const.”

20. Two Sum
function twoSum(nums, target) {
  const map = new Map();

  for (let i = 0; i < nums.length; i++) {
    const diff = target - nums[i];

    if (map.has(diff)) {
      return [map.get(diff), i];
    }

    map.set(nums[i], i);
  }
}

Explain:

“I use a hashmap to store values and indices. For each element, I check if the complement exists. This gives O(n) time complexity.”

21. Valid Parentheses
function isValid(s) {
  const stack = [];
  const map = { ')': '(', '}': '{', ']': '[' };

  for (let char of s) {
    if (!map[char]) {
      stack.push(char);
    } else {
      if (stack.pop() !== map[char]) return false;
    }
  }

  return stack.length === 0;
}
22. Remove duplicates (sorted array)
function removeDuplicates(nums) {
  let i = 0;

  for (let j = 1; j < nums.length; j++) {
    if (nums[i] !== nums[j]) {
      i++;
      nums[i] = nums[j];
    }
  }

  return i + 1;
}
23. Remove duplicates (unsorted)
function removeDuplicates(nums) {
  return [...new Set(nums)];
}
